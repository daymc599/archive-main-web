# archive-main-web

## 项目简介

`archive-main-web` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面集合的存储与分发点。适合存放各类可独立访问的 HTML 文档、演示页面、临时界面或历史快照。

## 目录结构

```
archive-main-web/
├── index.html          # 仓库首页（可选）
├── pages/              # 归档页面存放目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 共用资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有归档 HTML 文件，每个文件独立可访问。
- `assets/`：存放页面引用的共用静态资源，便于统一管理。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间无强依赖关系。每个页面可单独通过其路径访问。请勿在页面中引用外部不可控资源，以确保长期可访问性。

### 添加新页面

1. 将 HTML 文件放入 `pages/` 目录。
2. 如有共用资源，请放入 `assets/` 对应子目录。
3. 确保页面内资源引用路径正确（建议使用相对路径）。
4. 提交 Pull Request 或直接推送至主分支。

### 页面要求

- 使用标准 HTML5 语法。
- 避免使用外部 CDN 或第三方服务（除非必要）。
- 页面标题应简明描述内容。
- 建议添加适当的 meta 标签（如 `charset`、`viewport`）。

## 维护说明

- 本仓库由维护者不定期更新，接受合规的归档提交。
- 提交内容应不违反法律法规及 GitHub 服务条款。
- 维护者保留对归档内容进行审核、修改或删除的权利。
- 不建议在本仓库中存放动态内容、后台脚本或敏感信息。

## 贡献

欢迎通过 Issue 或 Pull Request 提出建议或提交新的归档页面。提交前请确保页面内容符合归档要求，并已测试可正常访问。
